video-studio · desktop app concept · Phase 0 (VS-78)

The pipeline, without the terminal.

Everything video-studio does today lives behind a command line. This is a guided macOS app that runs the same engine and the same tools underneath — so a musician or videographer can drop in footage, describe the cut they want, and walk out with a finished video.

Screen 04 — Design · the core idea

Two lanes, one engine

Auto — say what you want; Claude drives the pipeline and proposes a cut. Manual — open the timeline and place it yourself. They meet in the middle: Auto proposes, you refine in Review. Every auto decision is an editable artifact, so "AI did 90%, I tweaked 10%" is the normal path.

Screen 07 — Safety & permissions

The model proposes; the app decides

A permission layer the app owns — independent of Claude's own checks. Every action Claude wants is classified into something a person understands, and "always allow this kind" is a first-class choice that's remembered. Routine work stops nagging; genuinely risky actions still surface.

How it maps down: the always-safe categories are pre-approved via the SDK's allowedTools; everything else flows through canUseTool, where the app applies your remembered rules first and only shows this prompt on a miss. Categories are matched by type (any ffmpeg call), not exact command — so one "always" covers them all.
